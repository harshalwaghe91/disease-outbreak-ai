# OutbreakAI

AI-Based Disease Outbreak Prediction System Using Laboratory Test Data with a General Consultation Chatbot. This final-year project is a complete React, FastAPI, scikit-learn, and SQLite application.

> Academic decision-support prototype only. It is not a medical device, does not provide a diagnosis, and must not replace a qualified healthcare professional.

## Features

- Professional responsive healthcare SaaS landing page
- Random Forest prediction from demographics, symptoms, and 10 laboratory/vital inputs
- Low, Medium, or High risk, disease category, probability, and prevention guidance
- Analytics dashboard with Recharts visualizations
- Rule-based general consultation chatbot with emergency escalation
- SQLite-backed prediction report history
- Admin health and responsible-use dashboard
- Free-tier Vercel and Render deployment configuration

## Run locally, step by step

### 1. Prerequisites

Install Python 3.11+, Node.js 20+, and Git.

### 2. Clone and enter the project

```bash
git clone https://github.com/YOUR_USERNAME/disease-outbreak-ai.git
cd disease-outbreak-ai
```

### 3. Set up the backend

On Windows PowerShell:

```powershell
cd backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
cd ..
python model/train_model.py
cd backend
uvicorn app.main:app --reload
```

On macOS/Linux, activate with `source .venv/bin/activate`. The API runs at `http://localhost:8000`; interactive Swagger documentation is at `http://localhost:8000/docs`.

The training script creates 5,000 synthetic rows, trains disease and risk Random Forest pipelines, prints accuracy, precision, recall, and F1-score, and saves `model/outbreak_model.pkl`.

### 4. Set up the frontend

Open a second terminal:

```powershell
cd frontend
Copy-Item .env.example .env
npm install
npm run dev
```

The web app runs at `http://localhost:5173`.

### 5. Try the full flow

1. Open the prediction page and submit the prefilled demonstration values.
2. Review the model result and medical disclaimer.
3. Open Reports and Analytics to see the stored prediction.
4. Ask the chatbot about a common symptom.
5. Test an emergency phrase such as `chest pain` and confirm urgent-care guidance appears.

## API endpoints

| Method | Endpoint | Purpose |
|---|---|---|
| `POST` | `/predict` | Generate and store an outbreak risk prediction |
| `POST` | `/chat` | Return safe, rule-based general guidance |
| `GET` | `/dashboard` | Return dashboard summary counts |
| `GET` | `/reports` | Return recent prediction history |
| `GET` | `/analytics` | Return chart-ready aggregate data |
| `GET` | `/health` | Deployment health check |

## Model and metrics

The generated data intentionally simulates interpretable patterns: elevated WBC/CRP for bacterial infection, reduced oxygen saturation for respiratory disease, reduced platelets for dengue-like illness, and fever patterns for flu-like illness. This is synthetic data and cannot establish real-world clinical accuracy.

Each model is evaluated on a held-out 20% test set. The saved Joblib bundle includes:

- Accuracy
- Weighted precision
- Weighted recall
- Weighted F1-score
- Training timestamp and dataset size

To retrain:

```bash
python model/train_model.py
```

## Deploy for free

### 1. Push to GitHub

```bash
git init
git add .
git commit -m "Build OutbreakAI full-stack application"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/disease-outbreak-ai.git
git push -u origin main
```

### 2. Deploy the backend on Render

1. Sign in to [Render](https://render.com) and connect the GitHub repository.
2. Choose **New + в†’ Blueprint** and select the repository. Render detects `render.yaml`.
3. Set `FRONTEND_URL` initially to `http://localhost:5173`; update it after Vercel supplies the production URL.
4. Deploy and copy the backend URL, for example `https://outbreak-ai-api.onrender.com`.
5. Verify `https://YOUR_RENDER_URL/health` returns `{"status":"ok"}`.

Render's free filesystem is ephemeral, so SQLite report history can reset after redeployment or service replacement. For persistent production data, migrate to Render Postgres or another managed database.

### 3. Deploy the frontend on Vercel

1. Sign in to [Vercel](https://vercel.com) and import the same GitHub repository.
2. Set **Root Directory** to `frontend`.
3. Keep the detected Vite build command (`npm run build`) and output directory (`dist`).
4. Add environment variable `VITE_API_URL=https://YOUR_RENDER_URL` without a trailing slash.
5. Deploy and copy the Vercel domain.
6. In Render, change `FRONTEND_URL` to the complete Vercel URL and redeploy the backend.

### 4. Confirm production

- Load the Vercel landing page and navigate directly to `/dashboard` to verify SPA routing.
- Submit one prediction and confirm a report appears.
- Verify browser requests target Render and no CORS error appears.
- Confirm the chatbot emergency warning and disclaimer.

## Safety and limitations

- Do not enter names or personally identifying information.
- The training data is synthetic and the metrics measure synthetic test performance only.
- The probability is a model confidence/risk estimate, not disease prevalence.
- The chatbot is deterministic, general, and cannot consider full medical history.
- Emergency symptoms require immediate local emergency services, not chatbot interaction.
